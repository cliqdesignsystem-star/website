03 — Temperature & Mode

The two axes users control are temperature and brightness. These are independent settings that can be combined freely.

Setting Options What it controls
Temperature Warm / Cool The accent palette — warm gives orange/amber/terracotta, cool gives cobalt/blue/lilac
Mode Day / Night The background canvas — day uses Ivory, night uses Carbon Black (or Gunmetal for warm)

CSS Variable Mapping

All components consume semantic tokens only — never raw hex values. This makes theme-switching a single attribute change on <html> or <body>.

Token Role Used For
--background 60% Page background, section fills
--primary 30% Nav, CTAs, cards, prominent UI elements
--secondary 10% Accents, badges, highlights, borders
--tertiary optional 10% Extra accent when 60-10-10-10 split used
--foreground — All body text
--surface — Cards, modals, elevated containers

Switching Themes in Code

// Set theme via data-attribute on <html>
document.documentElement.setAttribute('data-theme', 'night-cool');

// Tailwind class usage (semantic tokens always)
<div className="bg-background text-foreground">
  <button className="bg-primary text-background">CTA</button>
  <span className="bg-secondary">Badge</span>
</div>

04 — Typography

Typography is split into three distinct roles: a proprietary display font for H1, a bold variable heading typeface for H2–H6, and a clean semi-bold sans-serif for all body copy.

H1 Display Font — ZtShago ExtraBold

ZtShago ExtraBold is a self-hosted woff2 file served from /fonts/ZtShago-ExtraBold.woff2. It is the flagship display font for all H1 hero headlines — bold, architectural, and unmistakeably B3YLAND. Fallback stack: Dela Gothic One → sans-serif.

H1 — ZtShago ExtraBold / falls back to Dela Gothic One
The Quick Brown Fox
Dela Gothic One — UX Fallback Only Dela Gothic One is not part of the brand type system. It is loaded exclusively as an immediate visual placeholder for H1 while the ZtShago woff2 file fetches from our own server. Once ZtShago has loaded and cached, Dela Gothic One is never rendered. Do not use it for any intentional typographic purpose.

H2 – H6 Heading Font — Phudu

Phudu (Google Fonts, variable weight 300–900) is used for all section headings H2 through H6. It is a geometric, condensed display typeface that pairs with ZtShago while remaining distinct from it. Used at font-weight: 900 and text-transform: uppercase. Fallback: sans-serif.

H2 — Phudu 900 / 42px
Design Meets Intention
H3 — Phudu 900 / 28px
Section Subheading
H4–H6 — Phudu 900 / 20px
Minor Heading Level

Body / UI Font — DM Sans

DM Sans is used for all body copy, UI labels, captions, inputs, and navigation text. It is loaded from Google Fonts as an optical-size variable font (9 to 40px). Default weight is 600 (Semibold). Bold passages use weight 700.

Body — DM Sans 600 / 16px / line-height 1.7

The body typeface is DM Sans at semibold (wght 600). This weight choice is deliberate: the B3YLAND warm palette — Harvest Orange, Amber Flame, and Ivory — has inherently low contrast on certain backgrounds. A semibold default ensures body text remains crisp and legible without relying on color alone. Bold elements (labels, captions) use wght 700.

DM Sans 700 — Bold / labels & emphasis

Bold text sample — for labels, strong emphasis, and UI metadata.

DM Sans 400 — Regular (available but not default)

Regular weight sample — avoid using as default body weight, use 600 instead for legibility.

Why DM Sans at Semibold (600)? The B3YLAND warm-palette themes use Ivory (#fdfdf1) as a background and Harvest Orange (#ff7900) as the primary color. These hues have reduced contrast ratios compared to classic dark-on-white setups. Setting the body font to font-weight: 600 by default compensates for this — text reads as confidently bold without needing to increase font size or add drop shadows. This is a deliberate accessibility and brand-voice decision: B3YLAND copy should feel assertive, never thin.

Character Infrastructure & L33T Rules

B3YLAND employs a strict character infrastructure for all display text. This ensures the brand remains distinct and technical in its presentation.

Rule 01 — Full Uppercase

All headers (H1 through H6) must be delivered in uppercase. No sentence casing is permitted for heading-level data.

Rule 02 — Highest Weight Alignment

Headings must always occupy the highest available font weight for their respective family. (ZtShago: 800 | Phudu: 900).

Rule 03 — L33T Replacement (H1 Only)

For all top-level H1 data, the letter 'E' is strictly replaced with the numeric '3' to mirror the brand wordmark architecture.

ARCHIT3CTING INFRASTRUCTUR3

05 — The 60-30-10 Rule

Every section of the website follows the classic interior-design color ratio. This creates visual hierarchy without chaos — the eye knows exactly where to look.

Standard Ratio — 60 · 30 · 10

60 Background
30 Primary
10 Secondary
Percentage Token Role Examples

30% --primary Main brand color Navigation, cards, primary CTA buttons, sidebars

Extended Ratio — 60 · 20 · 10 · 10

Some sections — particularly feature showcases, stat grids, or comparison layouts — benefit from a four-way color split. In these cases the dominant 60% canvas is supported by a 20% primary accent, followed by two 10% slices for secondary and tertiary highlights.

When to use 60-20-10-10 Use the 60-20-10-10 split for the most important "High Impact" sections (like the Hero) where all three brand accents work together to establish the visual hierarchy. This is the ultimate expression of the brand's character infrastructure.

Bullet Points & Separators

All bullet points used in multi-layer lists or feature grids must use the diamond symbol: ◆ (U+25C6). Standard dashes, dots, or ASCII bullets are prohibited for brand infrastructure lists.

Floating Cloud Effect

The white logo is used as an ambient site decoration — drifting "clouds" at randomised positions. Each cloud is a double-layer pair: a blurred shadow copy (opacity 6%, blur 6px, offset 6px) sits behind, and a brighter foreground copy (opacity 11%) sits on top. Both are semi-transparent. Clouds link to #work (case studies).

Cloud pair demo (on dark surface)

Shadow 6% + Foreground 11% — both semi-transparent

Pointer Event Model: The cloud wrapper div is pointer-events: none. Only the <a> anchor has pointer-events: auto, so clouds never block page interaction. Anchors use tabIndex={-1} to stay out of keyboard tab order.

09 — Images

Images in B3YLAND layouts follow a strict split ratio rule that prioritises content hierarchy. Two orientation modes exist: landscape (wider than tall) and portrait (taller than wide). Each maps to a different column ratio.

65 / 35 — Landscape Mode

For landscape photos (wide, cinematic, environment shots). The image occupies 65% of the row width, and the content section occupies the remaining 35%. Image is always placed adjacent to content, never below it on desktop.

LANDSCAPE — 65% IMAGE | 35% CONTENT
9.0765° N / 7.3986° E [CREATIVE_STUDIO]
Section Content Heading, body copy, CTAs

35 / 65 — Portrait Mode

For portrait photos (vertical, human-scale, candid). The image occupies 35% of the row width, and the content occupies 65%. Portrait images should never be stretched into landscape containers.

PORTRAIT — 35% IMAGE | 65% CONTENT
[FOUNDER_PORTRAIT]
Section Content Heading, body copy, CTAs

Image Overlay System

Every BrandImage has two data overlays rendered directly on the photo. These are always white text with varying opacity — never use brand colours on top of photos.

Overlay Position Style Example
GPS Coordinates Top-right Monospace, weight 400, opacity 55% 40.6892° N / 74.0445° W
Alt Label Bottom-left Monospace, weight 700, opacity 75%, uppercase [FOUNDERS_MEETING_ROOM] ← from alt="founders meeting room"
Alt Label Transformation Rule The alt attribute value is transformed at render time: spaces become underscores, all characters uppercased, wrapped in square brackets. Example: alt="system origin" → [SYSTEM_ORIGIN].

Gradient Scrim

A soft gradient scrim is applied over the image to ensure overlay legibility:

background: linear-gradient(
  to top,
  rgba(0,0,0,0.55) 0%,   /* bottom: alt label */
  rgba(0,0,0,0.00) 45%,  /* clear mid */
  rgba(0,0,0,0.28) 100%  /* top: coordinates */
);

Mobile Border Treatment

On mobile viewports, the image sits flush at the bottom of its section container. The three remaining sides (top, left, right) receive 2px solid borders. These borders must be theme-aware — using CSS variables to ensure correct contrast as the site switches between Warm and Cool palettes.

Border Edge Style Rule Implementation
Top + Right Border 2px Solid — Primary Accent var(--primary)
Left Border 2px Solid — Secondary Accent var(--secondary)
Bottom Border None (Flush to container) 0px / none

3. Theme Switcher (Client Component)

'use client';

const themes = ['day-warm', 'day-cool', 'night-warm', 'night-cool'];

export function ThemeSwitcher() {
  const setTheme = (theme: string) => {
    document.documentElement.setAttribute('data-theme', theme);
    localStorage.setItem('theme', theme);
  };

  return (
    <div>
      {/* Step 1: Brightness */}
      <button onClick={() => setTheme('day-warm')}>Day</button>
      <button onClick={() => setTheme('night-cool')}>Night</button>
      {/* Step 2: Temperature */}
      <button onClick={() => setTheme('day-warm')}>🔥 Warm</button>
      <button onClick={() => setTheme('day-cool')}>❄️ Cool</button>
    </div>
  );
}

Golden Rule Never hardcode a hex color in a component. Always use the semantic Tailwind tokens (bg-background, text-primary, etc.) so theme switching works automatically across all four themes without any extra logic.
Accessibility Note: var(--secondary) / Amber Flame
In the day-warm theme, the secondary color maps to Amber Flame (#ffb31f). Amber Flame does not have sufficient contrast against Ivory (var(--background)) or Harvest Orange (var(--primary)) to be legible at small sizes.

CRITICAL RULE: Never use var(--secondary) for body text, paragraphs, or small link labels (especially in the Footer). Reserve it strictly for large typography (like H1 tags or massive metrics) or thick borders where low contrast is stylized and acceptable.
